### 3.2 Typography System (v2.0)

#### Font Families

**Primary Font: Inter Variable**
- **Why:** Variable font = performance + flexibility
- **Weights used:** 400 (Regular), 500 (Medium), 600 (SemiBold), 700 (Bold)
- **Fallback:** system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif
- **Use:** All UI, headings, body text, navigation

**Monospace Font: JetBrains Mono**
- **Why:** Designed for code, excellent ligatures, clear distinction
- **Weights used:** 400 (Regular), 600 (SemiBold)
- **Fallback:** "Courier New", Courier, monospace
- **Use:** Code blocks, addresses, technical data, seed phrases

**Display Font (Optional): Syne**
- **Why:** Geometric, bold, attention-grabbing
- **Weight:** 700-800 (Bold-ExtraBold)
- **Use:** Hero headlines only, sparingly for impact
- **Fallback:** Inter if not loaded

## 4. TONE OF VOICE & CONTENT STRATEGY

### 4.1 Voice & Tone Framework

**Core Voice Attributes (Consistent across all content):**

| Attribute | Description | Example |
|-----------|-------------|---------|
| **Clear** | Simple language, no unnecessary jargon | "Bitcoin is money you control completely" NOT "Bitcoin is a decentralized peer-to-peer cryptographic protocol" |
| **Direct** | Get to the point, respect user's time | "Here's how to buy Bitcoin:" NOT "Before we explore the multifaceted world of purchasing digital assets..." |
| **Confident** | Authoritative without arrogance | "Bitcoin solves this problem" NOT "Bitcoin might potentially address this" |
| **Honest** | Acknowledge trade-offs and complexity | "This is complicated, but you can learn it" NOT "Bitcoin is simple!" |
| **Empowering** | Focus on what user CAN do | "You're in control" NOT "Let us handle it" |

**Tone Spectrum (Adjusts by context):**

```
Formal ‚Üê‚Äï‚Äï‚Äï‚Äï‚Äï‚Äï‚Äï‚Äï‚Üí Casual
  Legal     Education    Social
   Docs      Content     Media

Technical ‚Üê‚Äï‚Äï‚Äï‚Äï‚Äï‚Äï‚Äï‚Üí Simple
  Advanced   Beginner   Complete
   Guides    Guides     Newbie
```

### 4.2 Writing Guidelines

**DO:**
- Use active voice ("You control your Bitcoin" not "Bitcoin is controlled by you")
- Start with "why" before "how" (motivation before mechanics)
- Use analogies for complex concepts ("Bitcoin's supply cap is like digital gold")
- Break up long paragraphs (max 3-4 sentences per paragraph)
- Use bullet points and numbered lists liberally
- Include examples and real-world scenarios
- Define jargon the first time it appears
- Use "you" and "your" (second person, direct address)
- Cite sources for claims and data
- Acknowledge when something is your opinion vs. fact

**DON'T:**
- Use crypto slang excessively ("HODL," "To the moon," "WAGMI")
- Make price predictions or investment advice
- Use ALL CAPS for emphasis (use bold or italics)
- Write walls of text (break it up)
- Assume prior knowledge (or insult beginners)
- Use gendered language (use "they/their" or "you/your")
- Oversimplify to the point of inaccuracy
- Use fear-based language ("You'll lose everything if...")
- Make absolute statements when uncertain
